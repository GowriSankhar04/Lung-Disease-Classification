# 🫁 SafeBreath — AI-Powered Lung Disease Classification from Respiratory Sounds

SafeBreath is an AI-powered healthcare application that classifies respiratory diseases from lung sound recordings using machine learning and digital signal processing. It's designed to support early screening, healthcare education, and respiratory health awareness through an accessible mobile platform.

📲 **Live on Google Play:** [Play Store](https://play.google.com/store/apps/details?id=com.gs.safebreath)

## Overview

SafeBreath analyzes respiratory audio recordings — wheezes, crackles, and other abnormal breathing patterns — and classifies them into lung disease categories using an ensemble of handcrafted-feature machine learning models. The system combines digital signal processing with a cloud-hosted inference pipeline and a production Android app, live on the Google Play Store.

## Motivation

Respiratory diseases remain one of the leading causes of morbidity worldwide, and early identification often requires clinical expertise and specialized diagnostic equipment that isn't always accessible. Respiratory sounds carry valuable diagnostic signal, and SafeBreath aims to make that signal actionable through an intelligent system that analyzes respiratory audio and classifies lung disease risk directly through a mobile app.

## Dataset

The project uses publicly available respiratory sound datasets containing recordings from individuals with various pulmonary conditions.

**Dataset characteristics:**
- Respiratory sound recordings
- Multiple disease categories
- Normal and abnormal breathing sounds
- Different patient demographics
- Various recording conditions

The recordings include clinically relevant respiratory patterns associated with different lung diseases.

## Audio Signal Processing

Respiratory recordings go through extensive preprocessing and feature extraction before classification.

**Preprocessing steps:**
- Noise reduction
- Audio normalization
- Signal segmentation
- Feature standardization

**Feature extraction:**

Approximately **146 handcrafted audio features** are extracted using digital signal processing techniques, including:

- MFCCs (Mel-Frequency Cepstral Coefficients)
- Chroma features
- Spectral centroid
- Spectral bandwidth
- Spectral roll-off
- Zero crossing rate
- RMS energy
- Spectral contrast
- Tonnetz features
- Additional statistical descriptors

**Libraries used:** Librosa, NumPy, Scikit-learn

These features together provide a comprehensive representation of respiratory sound characteristics.

## Machine Learning Model

The classification system uses an **ensemble learning approach**, combining a Multi-Layer Perceptron (MLP) and a Random Forest Classifier.

Predictions from both models are combined to improve robustness and overall classification accuracy, yielding improved generalization, reduced overfitting, more stable predictions, and better performance across diverse respiratory sound samples compared to either model alone.

## Performance

The system achieved high classification accuracy across multiple lung disease categories, with:

- Reliable respiratory sound classification
- A robust feature extraction pipeline
- Real-time prediction capability
- Efficient deployment for mobile use

The ensemble architecture outperformed individual models in testing.

## Deployment

Trained models are deployed via **Hugging Face** for cloud-based inference:

```
Audio Recording → Feature Extraction → Hugging Face API → Ensemble Prediction → Mobile App Display
```

This keeps the mobile app lightweight, hosts inference in the cloud, scales easily, and allows model updates without requiring app changes.
